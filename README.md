# Learn Harness Engineering 4주 자율 스터디 (2026.06.29 ~ 2026.07.26)

`learn-harness-engineering` repo를 기준으로 4주 동안 진행하는 자율 스터디입니다.

* 강의 페이지: https://walkinglabs.github.io/learn-harness-engineering/ko/
* 기준 repo: https://github.com/walkinglabs/learn-harness-engineering

## 스터디 목적

AI 코딩 에이전트를 더 잘 이해하고 활용하기 위해 Harness Engineering의 개념을 학습합니다.

각자 매주 정해진 분량을 자유롭게 공부하고, 학습 내용을 Markdown 문서로 정리하여 PR로 공유합니다.

스터디 기간 동안 서로의 PR에 자유롭게 comment를 남기며 피드백합니다.

## 진행 방식

* 총 기간: 4주
* 매주 학습 분량: 챕터 3개
* 학습 방식: 자유 학습
* 주차 마무리: 매주 일요일 PR 생성
* Merge 시점: 4주 과정이 모두 끝난 뒤 일괄 merge
* PR 리뷰: 스터디 기간 내내 주차 상관없이 자유롭게 comment 작성

## 주차별 진행

| 주차  | 학습 분량 | 마감     |
| --- | ----: | ------ |
| 1주차 | 챕터 3개 | 일요일 PR |
| 2주차 | 챕터 3개 | 일요일 PR |
| 3주차 | 챕터 3개 | 일요일 PR |
| 4주차 | 챕터 3개 | 일요일 PR |

각 주차마다 학습할 챕터는 자유롭게 선택합니다.

## PR 제출 규칙

본인 nickname으로 된 branch를 {github-id}-week01 형식으로 만들어서 매주 일요일까지 PR을 올립니다.

PR에는 해당 주차에 학습한 3개의 강의 내용이 정리된 Markdown 파일 1개만 포함합니다.

### 제출 파일

* 파일 형식: `.md`
* 파일 개수: 1개
* 내용: 해당 주차에 학습한 3개 챕터 정리
* 작성 양식: 자유
* 분량: 자유

## 파일명 양식

root에 본인 github id로 폴더를 만들고, 해당 폴더 밑에 작성합니다.

```text
{github-id}/week01.md
{github-id}/week02.md
{github-id}/week03.md
{github-id}/week04.md
```

## PR 제목 예시

```text
[Week 01] Harness Engineering 학습 정리
```

```text
[Week 02] Learn Harness Engineering 정리
```

## Markdown 작성 예시

작성 양식은 자유입니다.
아래 형식은 참고용입니다.

```md
# Week 01 학습 정리

## 학습한 챕터

- Chapter 1. ...
- Chapter 2. ...
- Chapter 3. ...

## 핵심 내용 요약

### Chapter 1

...

### Chapter 2

...

### Chapter 3

...

## 인상 깊었던 내용

...

## 이해가 어려웠던 내용

...

## 추가로 찾아본 내용

...

## 스터디에서 이야기해보고 싶은 점

...
```

## 리뷰 / 피드백 방식

서로의 PR에 자유롭게 comment를 남깁니다.

예시:

* 이해가 잘 된 부분
* 설명이 더 필요한 부분
* 추가로 참고하면 좋을 자료
* 다른 관점의 해석
* 실제 코드나 프로젝트에 적용해볼 만한 아이디어
* 질문이나 토론하고 싶은 내용

리뷰는 필수가 아니라 자유 참여입니다.

## Merge 규칙

각 주차 PR은 바로 merge하지 않습니다.

4주차까지 모든 스터디가 끝난 뒤, 전체 PR을 확인하고 일괄 merge합니다.

## 스터디 원칙

* 정답보다 학습 과정 공유를 중요하게 봅니다.
* 문서 형식과 분량은 자유롭게 작성합니다.
* 완벽하게 정리하지 않아도 됩니다.
* 이해한 만큼 정리하고, 모르는 부분은 질문으로 남겨도 됩니다.
* 서로의 PR에 자유롭게 comment를 남기며 함께 학습합니다.

## 최종 목표

4주 동안 Harness Engineering의 주요 개념을 학습하고, AI 코딩 에이전트를 더 안정적으로 활용하기 위한 엔지니어링 관점을 익히는 것을 목표로 합니다.
